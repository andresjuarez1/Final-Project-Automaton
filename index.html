Togo Lang Gramática

Togo, es un lenguaje de programación pensado y diseñado para ser fácil de leer, escribir y entender, con el único propósito académico, para la asignatura de Leguajes y Autómatas de la carrera de IS de la Universidad Politécnica de Chiapas.

Está basado en lenguajes de programación como Go, Python y Javascript.

En esta sección verás cómo es la gramática de cuatro estructuras del lenguaje, y en el panel derecho, podrás escribir y probar el lenguaje.

Variables

La declaración de variables siguen la siguiente estructura:

type name: value;

Donde type puede ser algunos de los tipos de variable permitidas en Togo, como: num, str, bool.

Ahora, name hace referencia al nombre que se le asignará a la variable, este puede ser cualquier nombre alfanumérico, que no empiece con un dígito, por ejemplo: counter1 es un nombre de variable válido, sin embargo, 2counter? es un nombre de variable inválido, ya que contiene carácteres que no son alfanuméricos y empieza con un dígito.

Cada declaración de variable debe finalizar con punto y coma (;) para que pueda ser leido correctamente.

Tipo: str

Este tipo es para cadena de texto o carácteres únicos, es decir, únicamente admite asignación de tipo caracter alfanumerico dentro de dobles comillas ( "" ), como los siguientes ejemplos.

str name: "Raúl";
str ageAsString: "12";

Tipo: num

Este tipo admite únicamente valores numéricos, con o sin puntos decimales, como se muestran a continuación.

num numberOfCars: 12;
num salary: 123.09;

En caso de ser números decimales, el valor de la variable no debe empezar con un punto, en su lugar, debe ser colocado el valor de 0, seguido de los decimales que se desea representar, por ejemplo:

num goodVariable: 0.123;
num badVariable: .123;

En este caso, el editor mostrará un error en la linea 2 del código, ya que el valor de la definición de la variable numérica empieza con un punto.

Tipo: bool

Las variables de tipo booleana (bool) solo pueden tener dos valores: true y false, por lo que asignarle un valor distinto a este, le maracará un error de sintaxis.

Ejemplos

str name: "Pepe";
num age: 30.01;
bool married: true;

Funciones

La estructura principal de la funciones es la siguiente:

fn name(type param, ...):typereturn{ [code] }

El símbolo fn, es una palabra clave reservada, utilizada definir una función, mientras que name es el nombre de la función, que al igual que los nombres de variable, puede ser cualquier nombre alfanumérico, que no empiece con un dígito.

Posteriormente, se localizan los parámetros de la función, que van encerrados entre símbolos de parentesis (()), y dentro, los parámetros tienen la siguiente estructura: type name, donde type corresponde al tipo de parámetro que recibirá la función, que pueden ser los mismos tipos que tienen las variables: num, str y bool; y name, que corresponde al nombre del parámetro, que puede ser cualquiera, siempre que contenga únicamente carácteres alfanuméricos y noo empiece con dígito.

Una función puede o no tener parámetros al momento de su definició, así que una función puede verse de la siguiente forma: fn suma(int num1, int numb2):num{} o verse así: fn saludar():void{} y están definidas de forma correcta.

Despues de declarar los parámetros de la función, y de haber cerrado el símbolo de parentesis ()), llega la hora de definir el tipo de retorno de la función, es decir, que tipo de dato va a retornar la función tras su ejecución. Esto se define usando dos puntos (:), seguido del tipo, que en caso de la funciones, pueden ser las mismas que las usadas en la variables: num, str y bool, más un nuevo tipo vacío, identificado con la palabra clave void.

Finalmente, el contenido del código, representado en el ejemplo como [code], se escribe dentro de llaves ({}), el contenido puede ser cualquiera de las estructuras mostradas en este documentos, como otras funciones, variables, loops y/o condicionales.

Ejemplos

fn sumar(num a, num b):num { [code] }
fn saludar():void { [code] }
fn setPassword(str pwd):void { [code] }

Condicionales

Las condicionales en Togo tienen la siguiente estructura:

if var op var { [code] }

Donde if es la palabra clave reservada para definir una condicional. La clave var puede ser un nombre de varible declarada con anterioridad, un string o un número.

La referencia op hace referencia a los operadores lógicos admitidos en Togo, como pueden ser >, <, ==, !=.

Y por último, dentro de las llaves ({}) podrá ir cualquier otra instrucción.

Ejemplos

if age > limit { [code] }
if 12 == 15 { [code] }
if "algo" != "alga" { [code] }

Ciclo for

Togo permite una única forma te iteración, que es a través un ciclo for, el cual sigue la siguiente estructura:

for(iterable: initial, iterable op goal, iterable increment){ [code] }

Donde for es la palabra clave reservada para iniciar un ciclo, luego continua con la declaración de las variables iterables, que van dentro de paratesis (()), y en la referencia iterable, damos a entender el valor inicial de nuestra variable que será con la que se iterarás, esta pueda ser definida dentro del ciclo o únicamente llamada.

En caso de definirla dentro del ciclo, el tipo de variable se omite ya que el tipo num está implícito. Quedando un código como el siguiente:

for(alumno: 0, alumno < 30, alumno++){ [code] }

O bien, puedes poner el nombre de una sola variable, que ya haya sido declarada para usarla dentro del for, como se muestra a continuación:

for(alumno, alumno < 30, alumno++){ [code] }

Siguiendo con la estructura del for, tenemos la parte de iterable op goal, que hace referencia al valor donde el ciclo deberá detenerse, donde iterable es el valor que definimos como iterador inicial, op hace referencia al operador lógico, que son los mismos que usan dentro de las condicionales: >, <, ==, !=; y goal es el valor al cual se debe de comparar el iterable para que termine el ciclo, que puede ser otra variable, o un numero escrito, como se muestra en el siguiente segmento de código:

for(alumno, alumno < 30, alumno++){ [code] }

Luego continuamos con la sección de iterator increment, donde increment hace referencia a los operadores de incremento y decremento de Togo, que son ++ y --.

Ejemplos

for(foo: 1, foo < 5, foo++){ [code] }
